The Drush version manager is a free and convenient way to easily move through multiple Drush versions, with a straight to-the-point approach using a friendly command-line interface.

Tested with popular UNIX systems

Tested and used on GoDaddy Shared Linux Hosting and Cloud9. Being fully compatible with older versions of Bash will allow this to be installed anywhere!

Backwards compatibility

DVM has been tested and working on versions as low as 2.0.0, but we recommend 3.0.0 or later as it supports aliasing.

Symlinks

Use of UNIX symlinks to manage an unlimited number of versions with quick switching and inline execution of commands on any version.

Using another version

Use multiple versions for use with scripting with a simple command. A convenient way to check versions and run commands and scripts with logic.

Get the latest!

Conveniently update your local version or change to another version, choose to install/uninstall any specific version, or simply generate a list of versions all with simple commands.

Ansible

For convenient use with Ansible and Vagrant, an Ansible role is available for your automation needs. We encourage you to contribute to the repository should you run into any issues.

Made with love

DVM was recently rewritten in Go 1.8 in order to improve error handling, stdout reporting, compatibility and overall code cleanliness. DVM should basically work under any UNIX system because of this.

Simple requirements

DVM assumes you have composer installed, and if you're trying for versions prior to composer compatibility it will also assume you've got wget and unzip installed to the $PATH.

Completely free to use

DVM is supplied as is and without charge under the MIT license. Our hosting is taken care of by GitHub and we won't ever ask you for payment.

 

Why use DVM?

With the release of Drupal 8 and Drush 8, there are now many versions of Drush to choose from. With so many versions available, how does any one tie into your workflow?

DVM introduces the ability to quickly and simply execute drush scripts and commands using any version, anywhere, with support stretching all the way to versions 3.0.0 and above.

This project was inspired from similar projects such as NVM, RVM and PyEnv and of course the absence of a service which can provide the same function.
